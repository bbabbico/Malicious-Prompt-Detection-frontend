
======프로젝트 데이터 베이스======
밑에는 우리 프로젝트의 ERD 다이어그램임.
erDiagram
    USERS ||--o{ API_KEYS : "발급 (1:N)"
    API_KEYS ||--o{ DETECTION_LOGS : "사용됨 (1:N)"

    USERS {
        bigint user_id PK
        varchar email "로그인 ID"
        varchar password_hash 
        int daily_quota "일일 최대 호출량 (예: 1000)"
        int tps_limit "초당 최대 호출량 (예: 5)"
        timestamp created_at 
    }

    API_KEYS {
        bigint key_id PK
        bigint user_id FK
        varchar key_name 
        varchar key_prefix 
        varchar key_hash 
        timestamp expires_at 
    }

    DETECTION_LOGS {
        bigint log_id PK
        bigint key_id FK
        text raw_prompt 
        varchar used_track 
        float risk_score_pct 
        varchar action_taken 
    }

DB는 Mysql의 malicious-prompt-detection 이 스키마를 사용함, 이것도 도커로 따로 띄울거임. 위에 ERD 다이어그램을 참고하여 db를 구성함.

======회원 인증/인가======
우리 프로젝트는 FastAPI JWT 토큰 방식으로 회원 인증/인가를 처리하고,
/api/login , /api/signup 이 엔드포인트로 로그인 / 회원가입을 처리한다.
프론트에 이미 로그인과 회원가입 페이지가 만들어져 있으니, 백엔드 구성하고, 바로 연동하면 된다.

======API KEY======
